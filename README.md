## 📋 Overview

**GOFAP** *(Government Operations and Financial Accounting Platform)* — branded in its full form as **GOFAPS** *(Government Operations, Financial, Accounting & Personnel System)* — is a **100% configurable, cloud-native enterprise platform** purpose-built for government entities at every level: local, municipal, county, state, and federal.

GOFAPS functions as a **Complete Organizational Solution Technology (COST)**, unifying all core financial, operational, human resources, procurement, constituent services, and analytics capabilities into a single, integrated, compliance-ready platform. By replacing fragmented legacy tools with one extensible system, GOFAPS dramatically reduces administrative overhead, improves fiscal transparency, and enables government agencies to deliver better services to the public faster.

### Why GOFAPS?

| Challenge | GOFAPS Solution |
|---|---|
| Siloed financial systems | Unified general ledger, treasury, AP/AR, and payroll in one platform |
| Manual HR processes | End-to-end lifecycle from job posting through retirement |
| Slow procurement cycles | Automated solicitation, award, and contract management |
| Opaque budgets | Real-time dashboards with drill-down analytics |
| Disconnected citizen services | Configurable public portals per agency function |
| Compliance burden | Built-in FISMA, NIST 800-53, FedRAMP, GAAP, and Section 508 alignment |
| Outdated infrastructure | Cloud-native deployment on AWS, Azure, or on-prem |

## 🚀 Quick Start

### Prerequisites

| Requirement | Version |
|---|---|
| Node.js | 20+ |
| npm | 10+ |
| PostgreSQL | 15+ (or Neon serverless account) |
| Docker *(optional)* | 24+ |

### 1. Clone & Install

```bash
git clone https://github.com/UniversalStandards/GOFAP.git
cd GOFAP
npm install
```

### 2. Configure Environment

```bash
cp .env.example .env
# Edit .env with your credentials (see required variables below)
```

**Required Environment Variables:**

```env
# Database
DATABASE_URL=postgresql://user:password@host:5432/gofaps

# Authentication (Replit Auth / OIDC)
SESSION_SECRET=your-random-secret-min-32-chars
REPLIT_OIDC_ISSUER=https://replit.com/oidc
REPLIT_CLIENT_ID=your-client-id
REPLIT_CLIENT_SECRET=your-client-secret

# Payment Integrations (enable as needed)
STRIPE_SECRET_KEY=sk_live_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
MODERN_TREASURY_API_KEY=...
MODERN_TREASURY_ORGANIZATION_ID=...
UNIT_API_TOKEN=...
PAYPAL_CLIENT_ID=...
PAYPAL_CLIENT_SECRET=...

# AI Integration (optional)
ATLANTIS_AI_ENDPOINT=https://atlantis.spurs.gov/api
ATLANTIS_AI_KEY=...
```

### 3. Initialize Database

```bash
npm run db:migrate    # Run Drizzle migrations
npm run db:seed       # Seed sample data (development only)
```

### 4. Run Development Server

```bash
npm run dev
# Frontend: http://localhost:5173
# Backend API: http://localhost:3000
```

### 5. Run Tests

```bash
npm run test          # Run Vitest suite
npm run test:coverage # With coverage report
npm run typecheck     # TypeScript type checking
npm run lint          # ESLint
```

### Docker (Quickest Path)

```bash
# Development stack (app + postgres)
docker-compose up -d

# Production build
docker build -t gofaps:latest .
docker run -p 3000:3000 --env-file .env gofaps:latest
```

## 🔒 Security & Compliance

GOFAPS is architected to meet or exceed federal information security requirements:

| Standard | Status |
|---|---|
| **FISMA** (Federal Information Security Modernization Act) | ✅ Aligned |
| **NIST SP 800-53 Rev. 5** (Security and Privacy Controls) | ✅ Control mapping documented |
| **FedRAMP** (Federal Risk and Authorization Management Program) | 🔄 In progress |
| **FIPS 140-2** (Cryptographic Standards) | ✅ FIPS-compliant cipher suites |
| **GAAP / GASB** (Government Accounting Standards) | ✅ Compliant |
| **OMB Uniform Guidance** (2 CFR 200) | ✅ Grants module compliant |
| **Section 508 / WCAG 2.1 AA** (Accessibility) | ✅ Compliant |
| **PCI DSS Level 1** (Payment Card Security) | ✅ Compliant (Stripe-hosted card data) |
| **SOC 2 Type II** (Service Organization Controls) | 🔄 Audit scheduled |

### Security Controls

- **Zero-Trust Architecture**: Every request authenticated and authorized regardless of network origin
- **Encryption in Transit**: TLS 1.3 enforced; HSTS headers configured via NGINX
- **Encryption at Rest**: AES-256 for database, file storage, and backup encryption
- **Role-Based Access Control (RBAC)**: Granular permissions scoped to organization and module level
- **Organization-Level Data Isolation**: Strict row-level security preventing cross-tenant data access
- **Immutable Audit Log**: All financial transactions, data changes, and admin actions recorded with user, timestamp, and IP
- **Security Headers**: CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy via Helmet.js
- **Rate Limiting**: Per-IP and per-user rate limits on all API endpoints
- **Input Validation**: Zod schema validation on all API request bodies; parameterized queries throughout
- **CSRF Protection**: Double-submit cookie pattern for all state-changing operations
- **Secrets Management**: Environment-based secrets with fail-fast validation; no secrets in source control
- **Vulnerability Scanning**: GitHub CodeQL and Dependabot alerts enabled

For the full security policy and vulnerability reporting process, see [`SECURITY.md`](SECURITY.md).

## 🤝 Contributing

Contributions are welcome from both human and AI developers.

### Getting Started

1. Browse issues labeled [`ai-ready`](../../labels/ai-ready) or [`good-first-issue`](../../labels/good-first-issue)
2. Read the issue description and acceptance criteria thoroughly
3. Branch from `main` using the naming convention: `feat/issue-{number}-short-description`
4. Implement with tests (target ≥90% coverage for your changes)
5. Ensure `npm run lint`, `npm run typecheck`, and `npm run test` all pass
6. Submit a Pull Request referencing the issue

### AI Developer Guidelines

This project is explicitly designed for collaborative AI-assisted development:

- Prefer incremental, well-tested commits over large monolithic PRs
- All new API endpoints must: validate input (Zod), require authentication, log to audit trail
- Never ship mock data or placeholder returns on production paths — use feature flags or return HTTP 501
- Reference `DEVELOPMENT.md` for environment setup and coding conventions
- Reference `PRODUCTION_AUDIT.md` for known technical debt and outstanding issues

### Code Style

```bash
npm run lint          # ESLint (auto-fix with --fix)
npm run format        # Prettier formatting
npm run typecheck     # tsc --noEmit
```
